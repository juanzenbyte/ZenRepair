Proceso Recomendado

Paso 1 — Diagnóstico inicial
Antes de hacer nada, ejecuta primero el escaneo para identificar el tipo de problema. ¿Errores del sistema de archivos? ¿Sectores físicamente dañados? ¿La USB no aparece?

Paso 2 — Copia de seguridad
Si la USB es legible aunque sea parcialmente, copia todos los archivos que puedas a otro dispositivo antes de intentar cualquier reparación.

Paso 3 — Reparación
Aplica primero la reparación de errores del sistema de archivos. Es no destructiva y resuelve la mayoría de problemas comunes (tablas FAT corruptas, errores de directorio).

Paso 4 — Verificar sectores (si persiste)
Si la reparación no funciona, escanea los sectores dañados. Una USB con muchos bloques defectuosos es mejor reemplazarla.

Paso 5 — Formateo (último recurso)
Si nada funciona, formatea. Perderás los datos pero recuperarás la funcionalidad de la USB (si el daño no es físico).

Diagnóstico Rápido — Síntomas Comunes

🟡 "Es necesario formatear el disco antes de usarlo"
→ Sistema de archivos corrupto. Prueba chkdsk /f (Windows) o fsck (Linux) antes de formatear.
🟡 La USB aparece pero con 0 bytes o tamaño incorrecto
→ Tabla de particiones dañada. Usa diskpart (Windows) o fdisk/gparted (Linux) para recrear la partición.
🟢 La USB no aparece en el sistema
→ Prueba otro puerto USB, otro cable o en otro equipo. Si sigue sin aparecer, puede ser daño físico en el conector.
🔴 Escritura protegida / "El disco está protegido"
→ Windows: diskpart → attributes disk clear readonly. Linux: hdparm -r0 /dev/sdX
🔴 Errores constantes de lectura/escritura
→ Posible daño físico en la memoria flash. Si badblocks detecta muchos errores, reemplaza la USB.
💾 Herramientas de recuperación recomendadas
• TestDisk/PhotoRec — Recupera particiones y archivos (gratis, multiplataforma)
• Recuva — Interfaz gráfica, muy fácil (Windows, gratis)
• R-Studio — Profesional, recuperación avanzada
• ddrescue — Clona USBs dañadas sector a sector (Linux)
